#### 2019 年 10 月 - 現在: [DOGADOZO](http://dogadozo.com/) の新規開発

`サービス概要`  
放送局のアーカイブ動画販売プラットフォーム。

`チーム規模`  
自分含め 3 名の開発チーム。ビジネスサイド 1 名。

`担当領域`  
要件定義・設計・実装・テスト・リリース・保守を担当しました。

`使用技術`

- コード管理：Git, GitHub
- DB：PostgreSQL
- 言語・フレームワーク：Ruby(2.6.6), Ruby on Rails(6.0.3)
- テスティングフレームワーク：RSpec
- インフラ：Heroku
- AWS：
  - S3
  - CloudFront
  - Lambda
  - Elastic Transcoder
- 一部 CDN: imgix
- CI/CD：CircleCI

`発揮したバリュー`  
Facebook の広告データ取り込みや Elastic Transcoder を使った動画のトランスコードなど、社内・Web に情報が少ない実装に対しても粘り強く対処し実装してきました。これによって、Elastic Transcoder による動画変換処理・Carrierwave・S3 のオブジェクト生成をフックに AWS Lambda を起動、といった領域の知見を貯められたと思います。

プロダクト立ち上げ初期の段階から自動テスト・CICD を整備し、80-90％のカバレッジを維持してきました。これによって本番で大きなバグを起こさず、スピード感を持って開発を進めてこれたと思います。  
今まで社内になかった RSpec の知見を蓄積し、WOW U など DOGADOZO 以外のプロダクトにもテストを普及させていきました。

ビジネスサイドとの認識の齟齬を早期に潰すため、リリースのかなり前の段階でデモ動画を作成し共有するなどをしていました。これによってリリースの直前に設計を大きく変更する必要がある、といったことを避けられていたと思います。

## 強み

- 抽象的な要件を具体的に分解し、それらを実現するために必要な情報、技術を調査、習得しながらチームメンバーと協力しつつプロジェクトを推進する、といったことが得意だと思います。
- 「認識の差異によって大きな手戻りが発生する」「テストがないことでプロダクト開発が苦しくなる」など起こりうる悪い未来を考えて、それらに対して対策を考える、といったことが得意だと思います。
- 「プロダクトのコードを書く」という枠にあまりこだわず、自分が協力した方がうまく回りそうな領域にはよく入っていきます。
  - 例)Google Tag Manager を使ったコンバージョンの設定は、HTML の知識が必要なので自分でやってドキュメント化
  - 例)手動でやっていた映像制作会社の営業リスト作成をスクレイピングで自動化

## キャリア

ざっくりとした目標としては、30 代で新規プロダクトの技術選定に関与できるくらい技術に関する知識の広さと深さを持ちたいと考えています。少なくとも 30 歳くらいまではしっかりと手を動かすエンジニアとして働き、その過程で複数の技術を経験し視野を広げたいです。

手を動かしてものを作ることが好きなので、インフラ技術を専門とすることは現時点では視野に入れていません。またチームで色々考えながらものを作ることが好きなので、フリーランスという選択肢も現時点では考えていません。
